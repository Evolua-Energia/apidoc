Material de Migração da API

AS-IS

Antigamente a autenticação era feita diretamente no salesforce a partir dos dados de username, password, grant_type, client_id, client_secret a partir do POST:

curl --location --request POST 'https://evolua.my.salesforce.com/services/oauth2/token?username=email&password=password&grant_type=password&client_id=id&client_secret=secret' \'

TO-BE

A partir de hoje a autenticação será via Bearer, a partir apenas de um novo client_id e um client_secret disponibilizado pela Evolua, o exemplo de requisição está disponível tanto em (swagger) pela requisição Auth/Login:
PROD
HML
